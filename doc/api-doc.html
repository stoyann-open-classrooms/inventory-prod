Présentation de l'API

Bienvenue sur l'API de la Quincaillerie Calédonienne, un outil interne destiné à faciliter la gestion des stocks, des ventes, et des inventaires. Consultez les sections ci-contre pour plus d'informations sur les fonctionnalités disponibles.

Authentification

Cette API utilise une authentification basée sur des tokens pour sécuriser les accès. Vous devez inclure un token valide dans chaque requête.

Endpoints

Liste des principales routes disponibles :

  • GET / : Vérifie si l'API est en cours d'exécution.
  • POST /zones : Ajouter une nouvelle zone d'inventaire.
  • GET /zones : Récupérer toutes les zones d'inventaire.
  • PUT /zones/:id : Mettre à jour une zone d'inventaire.
  • DELETE /zones/:id : Supprimer une zone d'inventaire.
  • POST /agents : Ajouter un nouvel agent.
  • GET /agents : Récupérer tous les agents.
  • POST /records : Enregistrer une action (comptage, bipage, contrôle).
  • GET /inventory : Récupérer l'historique des inventaires.

Gestion des erreurs

Les erreurs sont renvoyées sous forme de réponses JSON contenant le code et le message d'erreur pour aider au débogage.

{
  "error": {
    "code": 400,
    "message": "Requête invalide. Veuillez vérifier les paramètres."
  }
}

Exemples de requêtes

Voici quelques exemples de requêtes courantes pour interagir avec l'API.

curl -X GET https://api.quincaillerie-caledonienne.com/zones
curl -X POST https://api.quincaillerie-caledonienne.com/records \
  -H "Content-Type: application/json" \
  -d '{"zone": "60f1b5c5f5d6f20015e0b567", "typeAction": "Comptage", "agent": "60f1b5c5f5d6f20015e0b678"}'
